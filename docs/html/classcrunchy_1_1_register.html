BOOL crunchy::Register::deregister_component ( bool  signedUID,
DWORD  keyLen,
std::string  path = TEMPVAR_PATH 
)

Deregisters component based upon component UID.

This will check if component is currently active or running, if TRUE; will throw error.

Attention
If component was registered using a DEFAULT_GENERATED_UID, this cannot deregister. If any other components use DEFAULT_GENERATED_UID it will deregister all of them.
Parameters
signedUID- has signed UID
keyLen- Key length
path- Grabs default tempfile path from global define
Returns
TRUE if component successfully deregistered. FALSE if component couldn't deregister

Main registry function for this file, this generates a default UID (if none is specified)

It checks for signed or unsigned UID's and can generate based off of the keySizeUID

For default UID signature see: DEFAULT_GENERATED_UID

Parameters
